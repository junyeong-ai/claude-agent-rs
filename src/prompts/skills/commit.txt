Analyze the staged git changes and create a well-formatted commit message.

Instructions:
1. Run `git status` to see all untracked and modified files
2. Run `git diff --staged` to see staged changes
3. Analyze the changes and determine the type (feat, fix, refactor, docs, test, chore)
4. Write a commit message following conventional commit format:
   - First line: type(scope): brief description (max 72 chars)
   - Empty line
   - Body: explain what and why (wrap at 72 chars)
   - Footer: co-authorship and generation note

Arguments: $ARGUMENTS

If arguments are provided, use them as hints for the commit message.
